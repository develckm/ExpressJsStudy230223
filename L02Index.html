express.js 미들웨어(==filter,interceptor) 개념 잡기!!

미들웨어는 요청 전에 가로채서 실행되는 동적페이지

  • /a.do 요청
  • /user/b.do 요청
  • /user/c.do 요청
  • /user/b.do?id=동홍 요청
  • /user/c.do?id=동일 요청
  • board/d.do 요청

미들웨어 정의 방법 및 개념

  • 정의 방법 : app.use("urlPattern",(req,res,next)=>{ ...next() })
  • urlPattern : * 와일드 카드를 이용해 요청의 복수를 한꺼번에 처리, 예) /a/b/* , *.do, /**/login.do
  • next : 브라우저가 요청한 url 을 미들웨어가 요청한 함수
  • app.use 를 순서대로 작성하면 미들웨어가 순서대로 처리된다.
  • 미들웨어는 중간 처리자로 특정 리소스에 대한 요청을 하기 전에 동작하는 동적페이지
  • 사용법 1 : 특정 url 에 대한 검사식 (로그인이 된 유저만 해당 리소스를 요청가능, 해당 파라미터가 꼭 존재해야함...)
  • 사용법 2 : 라이브러리를 미들웨어로 적용가능 (템플릿엔진,post 방식으로 오는 파라미터 처리, **)
